Build Your Own Slash Commands

The commands below are the starter set. The real unlock is making your own — turning any prompt you repeat more than twice into a single slash. Two routes. Always go the easiest first.

Option 1 — Simplest

Legacy Commands (Folder + Markdown)

Drop a markdown file in a folder, restart Claude, done. No YAML, no config, no skill structure. The fastest way to save a prompt.

  1. Create folder ~/.claude/commands/ (personal — works everywhere) or .claude/commands/ (project-scoped)
  2. Make a file: filename = command name. Example prospect-research.md becomes /prospect-research
  3. Put your prompt instructions inside the file as plain markdown
  4. Restart Claude Code (or the Claude desktop app)
  5. Type /prospect-research in chat
~/.claude/commands/prospect-research.md
↳  [prompt text here]
Option 2 — Recommended

Skills-Based (Folder + SKILL.md)

Skills let you version, share, and attach supporting files (templates, examples, data). Use this route when a command matters enough to reuse on a team.

  1. Create folder ~/.claude/skills/ (personal) or .claude/skills/ (project)
  2. Make one subfolder per skill — any name works
  3. Inside, add SKILL.md with YAML frontmatter (see right)
  4. Optionally add TEMPLATES.md, EXAMPLES.md, or data files in the same folder
  5. Web UI: zip the folder and upload under Settings → Features → Skills
---
name: prospect-research
description: Analyze a prospect
  and produce pain points,
  triggers, and objections.
model: claude-sonnet-4-6
---

Sales

/prospect-research

Background summary + pain points + buying triggers + likely objections from LinkedIn, website, and notes.

/prospect-research
Prospect: [NAME, TITLE, COMPANY]
Inputs: [PASTE LINKEDIN BIO, WEBSITE COPY, AND ANY CALL NOTES]
Produce:
1. 3-sentence background summary
2. Top 3 likely pain points (with why)
3. 3 buying triggers right now
4. 3 likely objections to my offer
5. 1 personalized opener I could use in outreach

/lead-qualification

Classify a lead as cold/warm/hot with discovery questions and a reply draft per temperature.

/lead-qualification
Lead: [NAME + COMPANY]
Signals so far: [PASTE EMAIL, DM, OR FORM RESPONSES]
Output:
1. Temperature: cold / warm / hot (with 2 reasons)
2. 5 discovery questions to move them forward
3. Reply draft tuned to their temperature (under 120 words)
4. Recommended next step + timing
