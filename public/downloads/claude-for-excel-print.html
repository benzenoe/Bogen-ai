Chapter 1

What Claude for Excel Is (and Who It's For)

Anthropic released Claude for Excel — a native Microsoft Excel add-in that runs Claude inside a sidebar. You press a shortcut, ask a question about your spreadsheet, and Claude responds with analysis, formulas, new tabs, pivot tables, or scenario runs. Every change is highlighted before it's applied, and every answer is cited back to the exact cell.

It went generally available on January 24, 2026 and is included at no extra cost on every paid Claude plan. The Free tier does not include it.

Who this guide is for

Realtors — if you run CMAs, track commission splits, model investment properties, or keep a rent roll in Excel, this tool replaces ~60% of the manual lift.

Entrepreneurs — if you live inside financial models, forecasts, or analytics dashboards, it shortens a half-day of cleanup into 20 minutes of review.

The mindset shift

Treat Claude like a brilliant junior analyst who has no context. You supply the business logic. It supplies the speed. Every output gets reviewed before you accept it.

Chapter 3

Sidebar Anatomy: What You're Looking At

When you press the shortcut, a pane slides in on the right side of Excel. Four things matter:

1. The chat input

Type natural-language requests. No formulas, no syntax. Plain English.

2. The slash menu ("/")

Type "/" in the input box to see Skills — reusable, one-click workflows. Your organization can build custom Skills for common tasks (see Chapter 7).

3. The change preview

Before Claude modifies your workbook, every change is highlighted. You review, then accept or reject.

4. Cell-level citations

When Claude answers, the answer links directly to the cells it referenced. Click the citation — Excel jumps to that cell.

The shortcut to memorize

Mac: Control + Option + C  ·  Windows: Control + Alt + C

Use it the way you use Command+S. Make it reflex.

Chapter 4

Five Prompt Patterns That Separate Pros From Tourists

Pattern 1 · Understand First

Before changing anything, have Claude map the workbook.

Walk me through this workbook. What's on each tab? How do they connect? Show me the data flow from inputs to final outputs.

Pattern 2 · Constrain Scope

Lock the edit zone so Claude doesn't touch what you didn't ask it to.

Update the Forecast tab only. Do not change anything in the Assumptions tab or formulas on any other sheet. Show me each change before applying.

Pattern 3 · Label Assumptions

Force Claude to flag what it's guessing.

Label every non-provided input as [ASSUMED] with your reasoning. What are the top 3 assumptions that, if wrong, would most change the output?

Pattern 4 · Safe Debug

Fix the root cause, not just the symptom.

There's a #REF! error in cell E15 on the Headcount tab. What's causing it, and what's the right fix? Explain the root cause before applying.

Pattern 5 · Scenario Stress Test

See the impact across the whole model, not just one cell.

Update the plan to assume 15% attrition instead of 10%. Show me every cell that changes, across all tabs, with before/after values.
Chapter 5

15 Copy-Paste Prompts for Entrepreneurs

Drop any of these into the sidebar. Tweak the bracketed placeholders to your spreadsheet.

1. Explain every formula in this tab with cell-level citations.
2. Find any circular references or broken links in this workbook and fix them.
3. Create a sensitivity analysis showing how revenue changes if price moves ±10% and volume moves ±5%.
4. Populate this template with the data from Sheet1 while keeping every formula intact.
5. Build a pivot table summarizing sales by region and product. Add a chart.
6. If we reduce costs by 20%, recalculate all dependent cells and show me the bottom-line impact.
7. Add data validation to the Assumptions tab so users can only enter values within reasonable ranges.
8. Apply conditional formatting to highlight every row where margin is below 20%.
9. Create a chart visualizing [range] as a [chart type]. Make it presentation-ready.
10. Trace every cell that depends on this assumption. What breaks if I change it?
11. Find and fix any #VALUE!, #DIV/0!, or #N/A errors in this workbook.
12. Generate a variance analysis comparing actual vs forecast across all 12 months.
13. Build formulas that dynamically calculate [metric] from inputs — no hardcoded values.
14. Clean up this dataset: normalize formatting, remove duplicates, fix inconsistent spelling.
15. Prepare this workbook for print: adjust column widths, hide intermediate calcs, format headers.
Chapter 6

10 Realtor Use Cases (The Anchor of This Guide)

Every use case below is a spreadsheet a working agent already has. Copy the prompt. Paste into the sidebar. Watch it build.

1
Comparative Market Analysis (CMA)
Manual comp entry, averages, and client-ready formatting eat 45 minutes per CMA.
I've entered 8 comparable properties in columns A–G (address, price, sqft, beds, baths, $/sqft, DOM, list date). Build a summary section that calculates avg price, median $/sqft, median DOM, and labels the market trend (up / flat / down) based on the last 90 days. Format it for a client presentation.
2
Commission Tracking & Splits
Team splits, overrides, and payouts are error-prone when done by hand.
I have a transaction log with: date, property, sale price, agent, side (listing/buyer). Build formulas that: calculate gross commission at 5.5%, split listing/buyer 60/40, apply [agent]'s override at [%] above base, track paid/pending, and show MTD + YTD totals per agent.
3
Listing Pipeline Funnel
Stages get updated inconsistently. Conversion rates are guesses.
Build a pipeline tracker with stages: lead, contacted, showing scheduled, showing completed, offer received, offer accepted, closed. Columns: date entered, days in stage, price, agent. Add formulas that auto-calculate conversion rates at each stage and a projected close rate for the quarter.
4
Lease vs. Buy Analysis
Buyers want the math. Most agents fake it with rules of thumb.
Build a lease vs. buy calculator for a $500K condo. Inputs: down payment %, mortgage rate, closing costs, annual HOA, taxes, insurance, maintenance; vs. annual rent with rent-growth %. Calculate 5-year NPV of each path, show the breakeven year, and recommend lease or buy with the reasoning.
5
Investment Property ROI
Cap rate, cash-on-cash, break-even occupancy: all spreadsheet math.
Set up an investment property analyzer. Inputs: purchase price, down payment, loan terms, monthly rent, vacancy rate, property mgmt fee %, capex reserve, insurance, taxes. Output: cap rate, cash-on-cash ROI, 5-year projected appreciation, break-even occupancy, and a ±10% rent sensitivity band.
6
Rent Roll Analysis
Multi-unit owners need a clear view of expirations and risk.
I have a 12-unit building with lease data (unit, tenant, rent, lease start, lease end, notice date). Build a rent roll showing upcoming expirations, notice deadlines, total monthly income, and a chart of lease-maturity profile by quarter. Flag any leases expiring in the next 60 days.
7
Transaction Coordination Tracker
Inspection, appraisal, underwriting, title, closing — one missed date = one dead deal.
Build a transaction tracker for 8 concurrent deals. Columns: property, buyer/seller, offer date, inspection due, appraisal due, clear-to-close date, closing date. Conditional formatting: red if due date − today ≤ 3, yellow if ≤ 7. Calculate days-to-close and identify the critical path.
8
Database Segmentation for Drip Campaigns
Hand-tagging 500 contacts is how weekends die.
I have a 500-contact database (name, email, phone, last contact, property interest, price range, neighborhood). Build a segmentation column that auto-tags: "expired-listing" if listed >180 days ago, "hot-prospect" if contacted within 30 days, "cold-lead" if >1 year. Add a pivot table with counts by segment.
9
Schedule C Expense Tracker
Agents leave thousands on the table every April because nothing's categorized.
Build an expense tracker with categories: advertising, car/mileage, office supplies, continuing education, meals (50% deductible), professional fees, technology. Auto-calculate subtotals by category and monthly/quarterly totals. Flag any meal expense > $50 for review. Output the total deductible amount.
10
Lead Scoring / Conversion Propensity
Calling leads in random order is a tax on your income.
My lead database has: last contact date, email open rate, click rate, meetings scheduled, response time, price range, timeline, prior transaction. Build a 1–100 lead score weighting recency 20%, engagement 30%, readiness 50%. Segment into hot / warm / cold and sort so I call the hottest first.
Chapter 7

Safety & Privacy: Know the Rules

Anthropic is explicit about what Claude for Excel can and can't safely process. Read this once, then follow it always.

Do

  • Review every change before accepting
  • Use consistent tab naming (Input_, Calc_, Output_)
  • Work on a copy for risky edits — keep a rollback
  • Validate AI-generated numbers against reality checks
  • Turn off model training in Settings → Privacy
  • Use range names for recurring inputs

Don't

  • Upload spreadsheets from untrusted senders
  • Paste full SSNs, DOBs, or raw PII unless compliant
  • Rely on Claude's numbers without spot-checking
  • Edit a live model — always work on a copy
  • Use the Free tier (no Excel access)
  • Assume confidentiality on Free/Pro for regulated data

Prompt injection risk

A malicious spreadsheet can hide instructions inside cell content that try to manipulate Claude's behavior. If a file came from outside your trusted network, treat it as suspicious. Strip or copy the data you need into a clean workbook before prompting.

Chapter 8

Your 7-Day Claude-for-Excel Implementation Plan

Don't try to learn everything at once. Run this seven-day sequence. By Sunday, you'll have replaced your five worst manual spreadsheet tasks.

Day 1 (Monday) — Install & first prompt

Install the add-in. Open any active spreadsheet. Run Pattern 1 ("Walk me through this workbook"). Note what Claude misses — that tells you how to document your files better.

Day 2 (Tuesday) — Your worst formula

Find the formula in one of your files that you've been afraid to touch. Ask Claude to explain it. Then ask Claude to rewrite it more cleanly, with comments.

Day 3 (Wednesday) — Your highest-impact realtor use case

Pick one use case from Chapter 6 that maps to your current week's work. Run the prompt. Save the output as a template you reuse.

Day 4 (Thursday) — Clean something ugly

Take a messy dataset — a list export, a CSV from Showingtime, your contact database. Run Prompt #14 (data cleanup). Compare before and after.

Day 5 (Friday) — A scenario test

In any financial model you rely on, run Pattern 5 (stress test). Try three scenarios. See what breaks.

Day 6 (Saturday) — Your Schedule C

Run Use Case #9. Categorize the year's expenses so far. This is the day that pays for the subscription.

Day 7 (Sunday) — Pick your Skill

From your week's work, identify the one prompt you typed more than once. That's your first custom Skill candidate. Save it.
